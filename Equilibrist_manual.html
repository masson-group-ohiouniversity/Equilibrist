Overview

Equilibrist is a browser-based scientific application built with Python and Streamlit. It runs locally on your machine. You define a chemical system via a plain-text script, simulate titration or time-resolved curves, and optionally fit binding or rate constants to experimental data.

Key capabilities

  • Arbitrary equilibrium networks — any number of species, reactions, stoichiometries, and multiple products per reaction.
  • Kinetics simulation — irreversible (>) and reversible (<>) kinetic steps, including pre-equilibria defined with = in the same script.
  • Free-energy input mode — reactions can be defined using ΔG° (van 't Hoff) and ΔG‡ (Eyring) energy terms in kcal/mol instead of K or k values, with a user-specified temperature.
  • Liquid and solid titrant modes — volume-based additions with dilution, or solid additions with fixed cell volume.
  • UV-Vis global spectral fitting — Beer–Lambert analysis over any wavelength range; molar absorptivities solved analytically at each step.
  • NMR fitting — fast-exchange chemical shift, slow-exchange integration, and simultaneous mixed-mode fitting. Available in both equilibrium and kinetics modes.
  • Concentration fitting — direct fitting of measured species concentrations.
  • Dual optimizer (concentration mode) — L-BFGS-B (fast, gradient-based) followed by Nelder-Mead (robust, gradient-free), user-selectable per fit run.
  • User-defined soft bounds on K — restrict the optimizer to physically meaningful equilibrium constant ranges.
  • Free energy table — after fitting, ΔG° (equilibrium) and ΔG‡ (rate constants, Eyring) are reported in kcal/mol at the script temperature.
  • Publication-quality figure export — one-click PDF with vector text, Arial font, for subsequent editing in Adobe Illustrator.
  • Thermodynamic cycle detection — automatic warning when declared equilibrium constants are mutually inconsistent.
  • One-click data export — timestamped Excel file with concentrations and variables, plus PNG snapshot.
  • Acid-base titration mode — $reactions acid-base section accepts pKa values directly; Kw, pH variable, and water equilibrium are injected automatically.
  • Outlier flagging — click any experimental data point to exclude it from fitting; hollow markers distinguish excluded from included points; excluded points are absent from publication PDFs.
  • In-app script editor — edit the script directly in the browser without leaving the app.

Installation

Equilibrist requires Python 3.10 or newer. Two installation methods are available: conda (recommended, especially on Windows) and pip.

Method A — conda (recommended)

Best if you already have Anaconda or Miniconda. If not, download Miniconda (free, ~50 MB) from docs.conda.io/en/latest/miniconda.html.

🪟

Windows

  1. Open Anaconda Prompt (installed with Miniconda).
  2. Navigate to the Equilibrist folder: cd C:\path\to\Equilibrist
  3. Create the environment (once): conda env create -f environment.yml
  4. Activate it: conda activate equilibrist
🍎

macOS / Linux

  1. Open Terminal.
  2. Navigate to the Equilibrist folder: cd ~/path/to/Equilibrist
  3. Create the environment (once): conda env create -f environment.yml
  4. Activate it: conda activate equilibrist
Why conda?

The environment.yml pins Python to 3.11 and installs all dependencies in an isolated environment — it won't conflict with any existing Python on your machine. Binary packages (scipy, numpy, matplotlib) install more reliably via conda on Windows.

Method B — pip

Best if you already have Python 3.10+ installed.

🪟

Windows

  1. Download Python from python.org/downloads. Tick "Add Python to PATH" during install.
  2. Open Windows Terminal or PowerShell.
  3. Navigate to the Equilibrist folder: cd C:\path\to\Equilibrist
  4. Install (once): pip install -r requirements.txt
🍎

macOS / Linux

  1. Python 3 is usually pre-installed. Verify: python3 --version
  2. Open Terminal and navigate to the Equilibrist folder.

Running Equilibrist

Open a terminal in the Equilibrist folder. If using conda, activate the environment first.

conda activate equilibrist     # conda users only — skip if using pip
streamlit run app.py

Streamlit starts a local server and opens your browser automatically at http://localhost:8501. The application runs entirely on your machine — no internet connection required after installation.

Every time you use Equilibrist

conda users: conda activate equilibrist then streamlit run app.py.
pip users: just streamlit run app.py.

Windows shortcut

Right-click the Desktop → New → Shortcut. Enter: cmd /k "conda activate equilibrist && streamlit run C:\full\path\to\app.py". Double-clicking launches Equilibrist in one step.

Press Ctrl + C in the terminal to stop the server.

Script Structure

An Equilibrist script is a plain-text file (any extension, typically .txt) that fully defines the chemical system. It is divided into named sections, each opened by a $ keyword on its own line. Section names are case-insensitive. The order of sections does not matter to the parser, but is preserved in the exported files.

Lines beginning with # are comments. Inline comments after # anywhere on a line are stripped before parsing. Comment lines are preserved verbatim in the exported files. The keyword $equilibria is accepted as a synonym for $reactions.

A minimal script:

$concentrations
G0 = 1.00 mM

$volumes
V0 = 500 uL

$titrant
Ht = 10.00 mM

$reactions
G + H = GH; log K1 = 4.0

$plot
xmax = 3.00
x = H0/G0
y = G, H, GH

Upload this file with the Equilibrist Script uploader in the left sidebar, or type it directly into the in-app script editor. The simulation runs immediately and re-runs on every sidebar change. Uploading a new script performs a full session reset automatically.

$concentrations

Defines the initial concentrations of all species present in the cell/NMR tube before titration or kinetics begins.

Name = value unit

Species name derivation

If the name ends in 0, the trailing zero is stripped to give the free-species symbol used by the solver. G0 → species G.

UnitMeaningNote
MMolar
mMMillimolarDefault when no unit given
uMMicromolar

All concentrations are stored and displayed in mM internally. Scientific notation is accepted (e.g. 1e-3 M).

$concentrations
G0   = 1.00 mM    # free-species name: G
Dye0 = 10 uM      # free-species name: Dye
Use in x-axis expression

The script name (e.g. G0) is available in the x-axis expression as the analytical concentration in mM — dilution-corrected at each titration point in solution mode, constant in solid mode. The corresponding titrant variable H0 (automatically derived from titrant key Ht) holds the total analytical concentration of H in mM. Their ratio H0/G0 equals the equivalents of H added — the dilution factors cancel exactly in solution mode, and in solid mode G0 is constant while H0 increases linearly with addition.

$volumes

Sets the initial cell/NMR tube volume. Only V0 is required.

$volumes
V0 = 500 uL
UnitMeaningNote
LLitres
mLMillilitresDefault when no unit given
uLMicrolitres

In solution mode, V₀ is the starting volume; dilution is recalculated at every titration point. In solid mode, V₀ is fixed throughout. $volumes is not required in kinetics mode (there is no titrant addition).

$titrant

Defines what is added to the cell. Two modes are available, selected on the header line. Not used in kinetics mode.

Solution mode (default)

Each line names a component and its stock concentration. Adding solution incrementally increases the cell volume, diluting all species.

$titrant
Ht = 10.00 mM   # primary titrant, 10 mM stock
Lt = 5.00  mM   # co-dissolved ligand (optional)

Solid mode

Use $titrant solid. No stock concentration is needed. Cell volume is fixed at V₀ throughout. An optional ratio after a semicolon sets stoichiometric fractions for mixed solid additions.

$titrant solid
Ht          # single solid component
$titrant solid
Ht; 1.0     # two-component mixture, mole ratio H : M = 1.0 : 1.3
Mt; 1.3

$temperature

Sets the temperature used for free-energy calculations. This affects two things: (1) conversion of ΔG values to K or k when using the free-energy input mode in $reactions, and (2) the ΔG° / ΔG‡ values shown in the Free energies table after fitting. The default is 298.15 K (25 °C) when the section is absent.

Three equivalent syntaxes are accepted:

$temperature
T = 25 C        # 25 °C  (recommended syntax)
$temperature
T = 310.15 K    # 310.15 K
$temperature 37 C   # inline — value and unit on the header line
UnitMeaning
KKelvin (default when no unit given)
CCelsius (converted to Kelvin internally)
Temperature and fitting

The temperature does not affect equilibrium or kinetics calculations — K and k values stored in the script are always the direct constants. Temperature only controls (a) the DG→K/k conversion at parse time, and (b) the ΔG display in the fit statistics panel.

$reactions

Defines all chemical reactions — equilibria and/or kinetics — in a single section. The arrow symbol determines the reaction type and the solver engaged.

ArrowTypeSolver
=Thermodynamic equilibriumEquilibrium solver
>Irreversible kinetic stepODE (scipy.integrate)
<>Reversible kinetic stepODE (scipy.integrate)
$equilibria alias

$equilibria is accepted as a full synonym for $reactions. Scripts using the old keyword remain valid.

Equilibrium reactions ( = )

reactants = products; log Kname = value

Parameter names (K1, K2, …) must be unique across the entire script. The value is log₁₀(K). You may also provide the linear K directly — Equilibrist converts automatically:

$reactions
G + H  = GH;  log K1 = 4.5       # log form
GH + H = GH2; K2 = 31623          # linear form, auto-converted
2 G + H  = G2H; log K3 = 5.1       # 2:1 stoichiometry

Stoichiometric coefficients are integers separated from the species name by a space: 2 G, 3 H. The space is mandatory — 2G without a space is interpreted as a species literally named 2G. Multiple products are allowed: G + H = GH_A + GH_B.

Soft bounds on equilibrium constants

You can restrict the optimizer to a physically meaningful range for each K by appending a from … to … clause. Bounds outside this range are penalised quadratically — the optimizer is strongly discouraged from leaving the region, but the constraint is not hard. The bound values follow the same convention as the K itself: log units if you wrote log K1, linear units if you wrote K1.

SyntaxMeaning
from lo to hiBoth lower and upper soft bounds
from loLower bound only
to hiUpper bound only
(absent)Default bounds: log K ∈ [−2, 14]
 
$reactions
L + Eu = EuL;  log K1 = 6.0;  from 2.0 to 12.0
L + Eu = EuL;  K1 = 1e6;     from 1e2 to 3.5e12  # linear bounds

Kinetic reactions ( > and <> )

As soon as any kinetic arrow appears in the script, Equilibrist switches to the ODE engine for the whole system. Rate constants can be entered in log form (log k1 = 3.0) or linear form (k1 = 1000). Both forms are accepted.

$reactions
A + B >  AB; log k1  = 3.0                  # irreversible, log form
A + B >  AB; k1  = 1000                   # irreversible, linear form
A + B <> AB; log k1  = 4.0; log k-1 = 1.0  # reversible
A + B <> AB; k1  = 2e4;  k-1 = 2.0         # reversible, linear

Mixed equilibria and kinetics

Both arrow types can coexist freely. Pre-equilibria (=) in a mixed system are enforced via large reversible rate constants consistent with the declared K value.

$reactions
G + H = GH;  log K1 = 4.0    # fast pre-equilibrium
GH    > P;   log k2 = -2.0   # slow irreversible step

Free-energy input mode (DG / DG#)

Instead of K or k values, reactions can be defined using free energies in kcal/mol. The temperature set in $temperature (default 298.15 K) is used for all conversions. Two prefixes are available:

PrefixMeaningUsed for
DG = valueReaction free energy ΔG° (kcal/mol), van 't Hoff: K = exp(−ΔG°/RT)Equilibrium reactions (=)
DG# = valueActivation free energy ΔG‡ (kcal/mol), Eyring: k = (k_BT/h) · exp(−ΔG‡/RT)Kinetic reactions (>, <>)

For reversible kinetic reactions (<>), both DG# (forward barrier) and DG (reaction energy) are required. The reverse barrier is computed as ΔG‡_rev = ΔG‡ − ΔG°. An error is raised if this would be ≤ 0 (unphysical: transition state below products).

DG and k/K inputs cannot be mixed on the same reaction line.

$temperature
T = 25 C

$reactions
# Equilibrium via ΔG° (van 't Hoff)
G + H = GH; DG = -5.5              # ΔG° = −5.5 kcal/mol → log K ≈ 4.04

# Irreversible kinetics via ΔG‡ (Eyring)
GH > P; DG# = 18.0                  # ΔG‡ = 18.0 kcal/mol

# Reversible kinetics via ΔG‡ and ΔG° (Eyring + van 't Hoff)
A + B <> C; DG# = 15.0; DG = -3.0  # ΔG‡_rev = 15.0 − (−3.0) = 18.0 kcal/mol
Parameter names in DG mode

When DG mode is used, parameter names are auto-generated: equilibrium constants are named K + label (e.g. K1), forward rate constants kf + label, reverse rate constants kr + label. An optional label can be appended to the DG prefix: DG#A generates kfA.

Parameter naming (general)

Parameter names can be any alphanumeric string — log K1, log Benzene, log beta3, k-1 are all equally valid. The distinction between an equilibrium constant and a rate constant is determined entirely by the arrow symbol (= vs > / <>), not by the name. Names must be unique across the entire script.

Acid-base mode ($reactions acid-base)

A dedicated section header for acid-base titration systems. Instead of writing out proton-transfer equilibria by hand, each line specifies an acid/base pair and its pKa. Equilibrist automatically builds the deprotonation reactions, injects the water autoionization equilibrium (Kw), and defines a pH variable.

Three line formats

FormatExampleNotes
Acid; pKaAcOH; 4.76Conjugate base auto-derived by stripping trailing H (AcOH → AcO). Acid name must end in H.
Acid; Base; pKaEt3NH; Et3N; 10.0Explicit conjugate base — required when the name does not end in H.
A; B; C; D; pKa1, pKa2, pKa3H3PO4; H2PO4; HPO4; PO4; 2.15, 7.20, 12.35Ladder form for polyprotic acids — species listed in deprotonation order; pKa values comma-separated as last field. Number of pKa values must equal number of species minus one.

Auto-injected elements (invisible to the user)

  • Water equilibrium — H2O = H + OH; Kw with logK = −14.0, added once.
  • Water concentration — H2O0 = 1 M understood as a unitary activity, hidden from the sidebar.
  • pH variable — pH = -log(H * 1e-3), available in $plot y and $variables. (Concentrations are in mM internally; the factor 1e-3 converts to M before taking the log.)

Sidebar behaviour in acid-base mode

  • The parameter section is headed pKa values instead of "Equilibrium constants".
  • Each pKa is displayed and entered as a positive number (e.g. 4.76); the solver receives logK = −pKa.
  • Kw is hidden — it is implicit and not user-editable.
  • H₂O is hidden from the concentrations list.

OH⁻ titrant

To titrate with base (NaOH, Et₃N, etc.), declare the titrant normally in $titrant. The OH species is already present via the Kw equilibrium — do not define it as a reaction.

$concentrations
CH3COOH0 = 1.0 mM

$volumes
V0 = 1.0 mL

$titrant
Et3Nt = 10 mM

$reactions acid-base
CH3COOH; CH3COO; 4.76   # explicit base name (AcO would need AcOH)
Et3NH; Et3N; 10.0         # amine: acid Et3NH, base Et3N

$plot
xmax = 3.0
x    = Et3N0/CH3COOH0
y    = pH
Solid acid-base mode

Use $reactions acid-base and combine with $titrant solid. Cell volume is fixed at V₀ throughout.

$nmr

Activates NMR fitting mode. One or both of the following keywords can appear in the section. Available in both equilibrium (titration x-axis) and kinetics (time x-axis) modes — the data format differs only in column A (see Kinetics Data Files).

shift: — Fast exchange (chemical shift titration)

The observed shift is a population-weighted average over all species. The target must be a species name or a $variables name whose expression gives the stoichiometric composition of the observed nucleus.

$nmr
shift: Gtot         # Gtot = G + GH + GH2 defined in $variables
shift: Gtot, Htot   # multiple independent signal targets

Data column headers must match the target name exactly, or use Name.N / Name_N suffixes for duplicate proton environments (e.g. Gtot.1, Gtot_2). The first data row sets δ_free for each column. Bound-state shift increments Δδ are solved analytically (linear least squares) inside each optimizer step; only log K (or log k) is optimised non-linearly.

integration: — Slow exchange (signal integrals)

Separate peaks for each species are integrated. Absolute concentrations are recovered using the host mass balance. The integers after the colon give the number of NMR-active nuclei contributing to each signal column, in order.

$nmr
integration: 3, 3, 6   # columns B, C, D: 3H, 3H, 6H respectively

Column headers should match species names (e.g. G, GH) optionally with .N or _N suffixes for duplicate peaks. The primary component (mass-balance denominator) is the first species listed in $concentrations.

Mixed mode (simultaneous slow + fast exchange)

Include both keywords in the same section. The first n data columns (where n = count of numbers in integration:) are integration columns; all remaining columns are chemical shift columns.

$nmr
integration: 3, 3   # columns B–C: slow-exchange integrals
shift: Gtot          # columns D onward: fast-exchange shifts

$spectra

Activates UV-Vis/circular dichroism/emission global spectral fitting using the Beer–Lambert law. Available in both equilibrium and kinetics modes. Three optional keywords may appear inside the section:

$spectra
transparent: H, Eu       # these species do not absorb
path = 0.2              # optical path length in cm (default 1.0)
read: cage               # read the spectrum of cage from sheet 2 of the data file
$spectra                 # valid: section with no body — all species may absorb

transparent / non-emissive

Species listed after transparent: are excluded from the absorber list and their molar absorptivity is fixed at ε = 0 at every wavelength. The synonym non-emissive: is also accepted (useful for fluorescence or CD experiments).

path = value

Sets the optical path length in cm. This value is used to convert the analytically solved absorptivity matrix from Beer–Lambert units (mM⁻¹, path absorbed) to true molar absorptivities ε (mM⁻¹ cm⁻¹) for display on the pure-species spectra plot and for export. It does not affect which constants are optimised; it is a display and export scaling factor only. Default: 1.0 cm.

read: — providing known spectra

Species listed after read: have their molar absorptivity ε(λ) provided by the user in a second sheet of the uploaded data file, rather than solved by the optimizer. Providing a known spectrum does not remove a species from the fitting problem — Equilibrist still uses Beer–Lambert to determine K. It simply constrains the ε of that species to the provided values, leaving fewer free parameters for the spectral decomposition.

$spectra
transparent: H
read: cage                # cage spectrum read from sheet 2
path = 1.0
$spectra
read: A, B, C             # multiple known spectra

Rules for read::

  • Species listed in read: but absent from sheet 2 of the data file are silently ignored — no error is raised. Only species that are both listed in read: and found in sheet 2 have their spectrum pinned.
  • Species listed in transparent: and in read: simultaneously remain transparent (ε = 0 everywhere). The transparent: designation takes priority.
  • Empty cells in sheet 2 (NaN values) are treated as "no data at this wavelength": ε is not pinned there and is solved freely instead. Empty cells are never interpreted as ε = 0. This means a partial spectrum (e.g., measured over 300–400 nm only, with the remaining wavelengths blank) pins ε only in the measured region and leaves the rest unconstrained.
  • Wavelengths in sheet 2 that fall outside the wavelength range of the experimental data are not used.

$spectra keywords summary

KeywordMeaningDefault
transparent: A, BSpecies with ε = 0 at all wavelengths. Also accepted: non-emissive: (synonym).None (all species may absorb)
path = valueOptical path length in cm. Scales ε display and export only; does not affect fitting.1.0 cm
read: A, BSpecies whose ε(λ) is read from sheet 2 of the data file. Species absent from sheet 2 are silently skipped. Empty cells in sheet 2 are treated as "no data" — ε is solved freely at those wavelengths.None (all absorber spectra solved)

All species not listed as transparent are treated as absorbers. When $spectra is present and a data file is uploaded, Equilibrist:

  • Displays the rainbow spectral plot below the main chart.
  • Updates xmax to the x-value of the last spectrum.
  • Shows the wavelength range controls in the sidebar, seeded from the file's actual wavelength extent.
Concentration fitting with spectra

Concentration fitting (the fit checkbox next to each concentration) is automatically disabled when only spectra data is loaded — concentrations are not identifiable from Beer–Lambert alone (the system A = C·ε is scale-invariant with respect to C₀).

$constraints

Defines inter-parameter relationships that the optimizer must respect during fitting. Each line is one constraint. The section is silently ignored during simulation — it only takes effect when the Activate constraints toggle is switched on in the sidebar.

Opt-in at fit time

Constraints are never active by default. The Activate constraints checkbox appears in the sidebar only when a $constraints section is present in the script. It is unchecked on every script load.

All modes support constraints: concentration, NMR (shift, integration, mixed), spectra, and kinetics (all variants). Equilibrium constants, forward rate constants, and reverse rate constants may all appear in the same constraint.

Log-space syntax (default)

When the expression contains only multiplication, division, or isolated parameter names — no addition or subtraction between parameters — Equilibrist evaluates everything in log space.

Written asMathematical meaning
K1 = K2K1 = K2
K1 / K2 = 1e3K1 / K2 = 1000
K1 * K2 = 1e8K1 · K2 = 10⁸
K1 * K2 / K3 = 1e5K1 · K2 / K3 = 10⁵
2 * K1 = K22 · K1 = K2 (linear factor)
2 * log K1 = log K2K2 = K1² (log-space exponent)
k1 / k-1 = 1e4k1 / k−1 = 10⁴ (hyphenated name)
log K1 - log K2 = 3K1 / K2 = 1000 (explicit log form)

$constraints
# Thermodynamic cycle: K3 must equal K1 * K2
K3 = K1 * K2

# Microscopic reversibility: on-rate / off-rate = equilibrium constant
k1 / k-1 = 1e4

# Ratio of binding constants within a factor of 10
1e-1 < K1 / K2 < 1e1

K-space syntax

When a + or − operator appears directly between two parameter names, Equilibrist switches to K-space evaluation: parameters are substituted as their actual K values (10logK).

$constraints
k1 + k2 = 1e7         # sum of two rate constants
2 * K1 + K2 = 3e6     # weighted sum

Hard vs soft enforcement

By default all constraints are soft: violations add a quadratic penalty to the objective. Appending ; hard increases the penalty weight by 10⁶, making violation effectively impossible in practice.

TypeInequality penalty weightEquality penalty weight
Soft (default)10⁶ × (violation)²10⁸ × (violation)²
Hard (; hard)10¹² × (violation)²10¹⁴ × (violation)²
 
$constraints
K1 = K2              # soft equality
K1 > 1e3; hard      # hard lower bound

Using the Activate constraints toggle

When a $constraints section is present, an Activate constraints checkbox appears at the bottom of the sidebar parameter list. It is unchecked by default on every script load.

Starting point reset

When you uncheck Activate constraints after a constrained fit, Equilibrist automatically resets the starting parameter values to the script defaults before the next fit. This prevents the unconstrained optimizer from starting at the constrained optimum — which may lie in a narrow valley — and failing to find the true global minimum.

Parameter errors reported after a constrained fit are computed from the data residuals only (constraint penalty excluded from the Hessian). This ensures reported uncertainties reflect genuine information content in the data.

Fitting

All modes minimise the sum of squared residuals (SSR) over the selected log₁₀(K) or log₁₀(k) parameters. Outlier-flagged points are excluded from the dataset before the fit begins — they contribute no residuals.

Re-fit loop (all NMR and kinetics modes)

The optimizer runs multiple Nelder-Mead passes in a loop. After each pass, parameter bounds shift to ±20% of the current fitted value ("bounds recentring"), and the simplex step size is set to the full feasible range per dimension. The loop continues until parameters stop moving (max change < tolerance) or the global timeout is reached. In integration and mixed modes, the back-calculated concentrations (bc) are frozen at the start of each pass to avoid a moving-target problem — bc is only recomputed between passes.

Concentration mode

The equilibrium is solved directly at each experimental x-point. Both L-BFGS-B and Nelder-Mead are available. When both are on, L-BFGS-B runs first; Nelder-Mead then refines from the best point found. In addition to log K values, initial concentrations and titrant stock concentrations can be fitted simultaneously by ticking their fit checkboxes in the sidebar.

NMR chemical shift mode

Mole fractions F_i(x) are fully determined for fixed K. Bound-state increments Δδ_i are solved analytically inside each step — only K is optimised non-linearly. Uses the re-fit loop with bounds recentring.

NMR integration mode

Back-calculated concentrations are derived from integrals using mass balances (K-independent). The optimizer moves K until theoretical curves match bc. Uses the re-fit loop with frozen bc per pass and bounds recentring.

NMR mixed mode

Integration and shift residuals combined into a single weighted objective. Integration bc is frozen per pass; shift SSR is recomputed each iteration. Uses the re-fit loop.

UV-Vis spectral mode

For fixed K, the concentration matrix C is fully determined. ε is solved analytically at each optimizer step: for species with no known spectrum ε is found by lstsq or NNLS; for species declared with read: whose spectrum was loaded from sheet 2, the corresponding ε rows are pinned to the provided values wavelength by wavelength (NaN entries in the known spectrum are treated as free). L-BFGS-B runs first, followed by Nelder-Mead. The optimizer selection checkboxes do not affect spectral fitting — both stages always run.

Kinetics modes

ODE integration (scipy) at each evaluation. All NMR and spectra fitting modes are available in kinetics mode, using time [s] as the x-axis instead of equivalents. The same re-fit loop applies.

Kinetics fitting speed

Each objective evaluation requires a full ODE integration (~20 ms on a typical machine). Fitting 3 or more parameters (e.g. k₁, G₀, H₀ simultaneously) may use the full timeout budget. This is expected — the optimizer tracks the best point seen and returns it when time expires, so a timed-out fit is still a valid fit. Increase the Timeout value in the sidebar if you want more optimizer time.

Soft bounds — applied as quadratic penalties outside [lo, hi]. Default range: log K ∈ [−2, 14] for equilibrium constants; no default bounds for rate constants.

UV-Vis timeout and identifiability

After every UV-Vis fit, Equilibrist analyses whether the fitted K values are uniquely identifiable by computing the condition number of C. If cond(C) > 10,000, a warning shows the condition number and the best-determined parameter combination (typically the cumulative product).

Auto-optimize wavelength range

When ticked, a two-pass procedure selects the most informative wavelength sub-range:

1

Full-range fit (pass 1)

Fits over the complete [min, max] range to get an initial ε(λ) estimate.

2

Range selection

Spectral contrast = variance of ε across absorbers at each wavelength. Edge wavelengths below the 10th percentile are trimmed. A 50 nm minimum window is enforced.

3

Refined fit (pass 2)

Re-fits on the selected range. Wavelength boxes update automatically.

Polyprotic acid — ladder syntax

Phosphoric acid titrated with NaOH. All three pKa values and all four species are defined in a single line using the ladder format. The titrant is OH (hydroxide); Equilibrist already knows OH from the auto-injected Kw equilibrium.

$concentrations
H3PO40 = 1.0 mM

$volumes
V0 = 1.0 mL

$titrant
OHt = 10 mM

$reactions acid-base
H3PO4; H2PO4; HPO4; PO4; 2.15, 7.20, 12.35   # ladder: 4 species, 3 pKa values

$plot
xmax = 3.0
x    = OH0/H3PO40
y    = pH
What Equilibrist injects automatically

The water equilibrium (H2O = H + OH; logKw = −14), the water concentration (H2O0 = 1 M), and the pH = -log(H * 1e-3) variable are all injected silently. The sidebar shows pKa values as positive numbers. Kw and H₂O are hidden. Fitted pKa values are written back to the sidebar immediately after each fit.

Example: Constraints

Equilibrium: thermodynamic cycle

$concentrations
A0 = 1.0 mM
B0 = 1.0 mM

$volumes
V0 = 500 uL

$titrant
Bt = 10.0 mM

$reactions
A     + B  = AB;  log K1 = 5.0
AB    + B  = AB2; log K2 = 4.0
A  + 2 B  = AB2; log K3 = 9.0   # must equal K1 * K2

$constraints
K3 = K1 * K2

$plot
xmax = 3.00
y    = A, B, AB, AB2

Kinetics: microscopic reversibility

$concentrations
A0 = 50 uM
B0 = 30 uM

$reactions
A + B <> C; k1  = 2e4; k-1 = 2.0
C       >  D; k2  = 1.5

$constraints
k1 / k-1 = 1e4   # microscopic reversibility

$plot
xmax = 3.00
y    = A, B, C, D

$spectra
transparent: B
How to use constraints

Upload the script, load experimental data, check the parameters to fit. The Activate constraints checkbox appears at the bottom of the parameter list in the sidebar. Run once unconstrained to see the free fit, then check the box and re-run to enforce the relationship.
